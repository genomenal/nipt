# Запуски

Блок "Запуски" включает следующие страницы:
- "***Запуски***" - содержит все добавленные в систему запуски и их данные. Запуски подробно описаны
в [соответствущем разделе](/runs).

Если у вас ещё нет добавленных запусков, то страница "Запуски" выглядит так:

<p align="center">
<img src={require('/img/rus/no_runs_page.png').default} width="1000"/>
</p>

Страница с добавленными запусками выглядит следующим образом:

<p align="center">
<img src={require('/img/rus/runs_page.png').default} width="1000"/>
</p>

Вверху страницы находится кнопка <img src={require('/img/runs/add_run.png').default} width="150"/>, при нажатии на
которую открывается окно <a href="/runs">создания запуска</a>.

- “***Архив***” - содержит те запуски, данные которых были архивированы. Вы можете перенести сюда
неактуальные данные или данные, которые вы хотите удалить. Запуски, находящиеся в "Архиве", не отображаются
на странице "Запуски".

<p align="center">
<img src={require('/img/rus/run_archive.png').default} width="1000"/>
</p>

Вверху страниц блока находится окно поиска, позволяющее искать запуски по названию или описанию:
<p align="center"><img src={require('/img/rus/search_run.png').default} width="450"/></p>

Внизу страниц блока находятся:
- Информация о количестве запусков: порядок запусков, отображённых на странице, (максимум 10 запусков может
быть показано на странице, так что порядок будет таким: 1-10, 11-20 и т.д.) и суммарное количество запусков.
- Компонент пагинации: <img src={require('/img/rus/pagination.png').default} width="400"/>, который позволяет
перемещаться между страницами с запусками. Номер открытой страницы выделен синим кружком. Если нажать на этот
номер, то страница перезагрузится. Чтобы переместиться на страницу с конкретным номером, либо нажмите на этот номер,
либо нажмите на <img src={require('/img/rus/three_points.png').default} width="30"/>, введите нужный номер и нажмите
клавишу ввода или кликните вне области заполнения значения.
Чтобы переместиться на предыдущую по номеру страницу,
нажмите на <img src={require('/img/rus/back.png').default} width="30"/>, а чтобы открыть следующую -
нажмите на <img src={require('/img/rus/next.png').default} width="30"/>.

### Колонки {#runs-columns}

На страницах блока "Запуски" информация представлена в виде таблицы со следующими колонками:

1. **ЗАПУСК** - названия созданных запусков;
2. **СТАТУС ОБРАЗЦА** - информация о количестве образцов, находящихся в определенном статусе,
для каждого запуска:<br></br><img src={require('/img/rus/in_process_status.png').default} width="22"/> - образцы
запуска в
процессе обработки;<br></br><img src={require('/img/rus/queed_status.png').default} width="14"/> - образцы запуска,
стоящие в очереди на обработку;<br></br><img src={require('/img/rus/success_status.png').default} width="22"/> - успешно
обработанные образцы запуска;<br></br><img src={require('/img/rus/problems_status.png').default} width="24"/> - образцы
запуска, обработанные с
некритичными проблемами;<br></br><img src={require('/img/rus/failed_status.png').default} width="20"/> - образцы
запуска, обработанные с ошибкой;
3. **СОЗДАН** - дата добавления запуска в формате д.М.гггг;
4. **ИЗМЕНЕН** - дата последних изменений в данных запуска в формате д.М.гггг.

### Сортировка {#runs-sorting}

По умолчанию таблица отсортирована от запусков, добавленных последними, к запускам, добавленным первыми,
(т.е. по убыванию значений в колонке "Создан"). Таблицу можно отсортировать по возрастанию или убыванию
значений следующих колонок: "Запуск", "Создан", "Изменен".
Для сортировки таблицы по значениям колонки нажмите на её название (нажмите один раз, чтобы отсортировать
по возрастанию, или два раза, чтобы отсортировать по убыванию). После этого у названия колонки появится
стрелка <img src={require('/img/vv/sort_asc.png').default} width="17"/> (при сортировке по возрастанию)
или <img src={require('/img/vv/sort_desc.png').default} width="17"/> (при сортировке по убыванию).
Для сброса сортировки обновите страницу браузера. Колонка "Запуск" сортируется по алфавиту,
а колонки "Создан" и "Изменен" - от наиболее ранней к наиболее поздней дате и наоборот.

### Архивация, восстановление и удаление запусков {#delete-run}

Для ***архивации запуска*** и его образцов наведите курсор на строку запуска на странице "*Запуски*" и
нажмите на появившийся активный элемент <img src={require('/img/rus/delete_btn.png').default} width="18"/>.
Запуск будет перемещен на страницу "*Архив*". Перейдя на эту страницу, вы можете:

1. ***Восстановить*** данные запуска, наведя курсор на строку запуска и нажав на появившийся активный
элемент <img src={require('/img/rus/restore_btn.png').default} width="22"/>.
Тогда запуск будет перемещен обратно на страницу "*Запуски*".

2. ***Удалить запуск*** и его образцы, наведя курсор на строку запуска и нажав на появившийся активный
элемент <img src={require('/img/rus/delete_btn.png').default} width="18"/>.
Вы увидите окно подтверждения удаления запуска:

<p align="center">
<img src={require('/img/runs/confirmation_run.png').default} width="380"/>
</p>

Для подтверждения удаления запуска нажмите <img src={require('/img/runs/confirm.png').default} width="150"/>.
Для отмены удаления запуска нажмите <img src={require('/img/report_templates/cancel.png').default} width="150"/>
или вне окна подтверждения.

Кроме того, вы можете удалить все архивированные запуски,
нажав на <img src={require('/img/runs/clear_archive_btn.png').default} width="170"/> на странице "*Архив*".
