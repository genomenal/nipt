# Образцы

Страница "Образцы" - начальная страница NIPT Helicon, которая включает все загруженные в систему образцы, которые
принадлежат пациентам или запускам, не находящимся в архиве. Если для образца определён как пациент, так и запуск,
то образец будет отображаться на странице "Образцы", если хотя бы одна из этих сущностей не архивирована.

Если вы не пользовались NIPT Helicon раньше и у вас нет загруженных образцов, то страница будет выглядеть следующим
образом:

<p align="center">
<img src={require('/img/rus/empty_sample_page.png').default} width="1000"/>
</p>

Впоследствии вы увидите здесь все образцы, находящиеся в этом аккаунте и принадлежащие неархивированным пациентам
и/или запускам:

<p align="center">
<img src={require('/img/rus/samples_page.png').default} width="1000"/>
</p>

Вверху страницы находятся:
- Окно поиска, позволяющее искать образцы по имени файла и по коду
пациента или названию запуска, которому они принадлежат:
<p align="center"><img src={require('/img/rus/search_sample.png').default} width="500"/></p>
- Кнопка <a href="/add-patient">добавления пациента</a>
в систему <img src={require('/img/rus/add_patient_btn_from_samples.png').default} width="180"/>.
- Кнопка <a href="/runs">добавления запуска</a>
в систему <img src={require('/img/rus/add_run_btn.png').default} width="170"/>.

Внизу страницы находятся:
- Информация о количестве образцов: порядок образцов, отображённых на странице, (максимум 10 образцов может
быть показано на странице, так что порядок будет таким: 1-10, 11-20 и т.д.) и суммарное количество образцов.
- Компонент пагинации: <img src={require('/img/rus/pagination.png').default} width="400"/>, который позволяет
перемещаться между страницами с образцами. Номер открытой страницы выделен синим кружком. Если нажать на этот
номер, то страница перезагрузится. Чтобы переместиться на страницу с конкретным номером, либо нажмите на этот номер,
либо нажмите на <img src={require('/img/rus/three_points.png').default} width="30"/>, введите нужный номер и нажмите
клавишу ввода или кликните вне области заполнения значения.
Чтобы переместиться на предыдущую по номеру страницу,
нажмите на <img src={require('/img/rus/back.png').default} width="30"/>, а чтобы открыть следующую -
нажмите на <img src={require('/img/rus/next.png').default} width="30"/>.

### Колонки {#sample-columns}

На странице "Образцы" информация представлена в виде таблицы со следующими колонками:
1. **ОБРАЗЕЦ** - информация о загруженных в систему образцах, которые принадлежат пациентам или
запускам, не находящимся в архиве:
- **Статус обработки образца**:<br></br><img src={require('/img/rus/normal_in_process.png').default} width="18"/> - образец
в процессе обработки или в
очереди на обработку;<br></br><img src={require('/img/rus/normal_success.png').default} width="18"/> - успешно
обработанный образец;<br></br><img src={require('/img/rus/normal_problems.png').default} width="18"/> - образец,
обработанный с
некритичными проблемами;<br></br><img src={require('/img/rus/normal_failed.png').default} width="18"/> - образец,
обработанный с ошибкой.
- **Имя файла образца** без расширения и постфиксов (если файлы названы
согласно <a href="https://support.illumina.com/help/BaseSpace_Sequence_Hub_OLH_009008_2/Source/Informatics/BS/NamingConvention_FASTQ-files-swBS.htm">конвенции Illumina</a>).
В случае парного секвенирования приводится только имя первичного файла.
Нажав на имя файла, можно открыть <a href="/results/results">страницу образца</a>.
- **Формат файла**: <i>FASTQ</i>.
- **Размер образца**. В случае парного секвенирования будет отображен суммарный размер файлов образца.
2. **ПАЦИЕНТ / ЗАПУСК** - код пациента (обозначен <img src={require('/img/rus/patient_icon.png').default} width="20"/>)
и/или название запуска (обозначен <img src={require('/img/rus/run_icon.png').default} width="20"/>),
которым принадлежит образец. Нажав на код пациента или название запуска, можно
открыть, соответственно, <a href="/add-patient#patient-page">страницу пациента</a>
или <a href="/runs#run-page">страницу запуска</a>.
3. **СОЗДАН** - дата начала загрузки данных образца в систему в формате д.М.гггг;
4. **ИЗМЕНЕН** - дата последних изменений в данных образца в формате д.М.гггг.

### Сортировка {#samples-sorting}

По умолчанию таблица отсортирована от образцов, загруженных последними, к образцам, загруженным первыми,
(т.е. по убыванию значений в колонке "Создан"). Таблицу можно отсортировать по возрастанию или убыванию
значений всех её колонок.
Для сортировки таблицы по значениям колонки нажмите на её название (нажмите один раз, чтобы отсортировать
по возрастанию, или два раза, чтобы отсортировать по убыванию). После этого у названия колонки появится
стрелка <img src={require('/img/vv/sort_asc.png').default} width="17"/> (при сортировке по возрастанию)
или <img src={require('/img/vv/sort_desc.png').default} width="17"/> (при сортировке по убыванию).
Для сброса сортировки обновите страницу браузера. Колонки "Образец", "Пациент" и "Запуск" сортируются по алфавиту, а
колонки "Создан" и "Изменен" - от наиболее ранней к наиболее поздней дате и наоборот.

### Удаление образца {#delete-sample}

Чтобы удалить образец из системы, наведите курсор на строку образца и нажмите
на <img src={require('/img/rus/delete_btn.png').default} width="20"/>. Появится окно подтверждения удаления:

<p align="center">
<img src={require('/img/rus/delete_sample.png').default} width="450"/>
</p>

Для подтверждения удаления образца нажмите <img src={require('/img/rus/delete.png').default} width="150"/>.
После этого образец удалится как со страницы "Образцы", так и со страницы пациента и/или запуска, которому он
принадлежал.
