# Администрирование

Блок "Администрирование" доступен только для администратора Программы.
Он включает страницы ["Конфигурации"](/overview/administration#configurations),
["Пользователи"](/overview/administration#users) и
["Активные задачи"](/overview/administration#active-tasks).

### Конфигурации {#configurations}

Страница содержит конфигурации настроек, которые можно передать новым пользователям при регистрации их
в Системе.

<p align="center">
<img src={require('/img/rus/configurations_page.png').default} width="900"/>
</p>

Чтобы добавить конфигурацию в Систему, нажмите
на кнопку <img src={require('/img/rus/add_configuration.png').default} width="200"/>. В открывшемся окне
создания конфигурации введите имя и описание (по желанию) конфигурации:

<p align="center">
<img src={require('/img/rus/add_configuration_form.png').default} width="450"/>
</p>

Затем добавьте в конфигурацию предварительную установку настроек и/или шаблон отчёта, которые вы хотите передать
новому пользователю.

Чтобы добавить предустановку настроек в конфигурацию, нажмите на соответствущее поле и выберите
предустановку среди [предустановок, добавленных в вашу Систему](/settings/settings/settings). Затем
нажмите на <img src={require('/img/rus/add_btn.png').default} width="110"/>. Одну из добавляемых
предустановок необходимо установить предустановкой по умолчанию; выбор можно поменять,
нажав на <img src={require('/img/rus/default_btn.png').default} width="30"/> у необходимой предустановки.
Для удаления предустановки из конфигурации
нажмите на <img src={require('/img/rus/delete_btn_white.png').default} width="30"/>.

<p align="center">
<img src={require('/img/rus/settings_in_configuration.png').default} width="500"/>
</p>

Чтобы добавить шаблон отчёта в конфигурацию, нажмите на соответствущее поле и выберите
шаблон среди [шаблонов, добавленных в вашу Систему](/settings/report-templates/report-templates). Затем
нажмите на <img src={require('/img/rus/add_btn.png').default} width="110"/>.
Для удаления шаблона из конфигурации
нажмите на <img src={require('/img/rus/delete_btn_white.png').default} width="30"/>.

После того, как вы добавили в конфигурацию хотя бы одну предустановку настроек или хотя бы один
шаблон отчёта, кнопка <img src={require('/img/rus/create_btn.png').default} width="170"/>
станет активной. Нажмите на неё для добавления конфигурации в Систему.

Чтобы внести изменения в конфигурацию, нажмите на её строку в таблице, внесите изменения в окне
и нажмите <img src={require('/img/rus/create_btn.png').default} width="170"/>.

Добавленные конфигурации представлены на странице в виде таблицы со следующими колонками:
1. **КОНФИГУРАЦИЯ** - имя конфигурации;
2. **ОПИСАНИЕ** - описание конфигурации;
3. **ССЫЛКА ДЛЯ РЕГИСТРАЦИИ** - колонка, в которой содержится ссылка для регистрации новых пользователей
в Системе с данной конфигурацией. Чтобы скопировать ссылку,
нажмите на кнопку <img src={require('/img/rus/copy_to_clipboard.png').default} width="30"/>.
4. **ИЗМЕНЕН** - дата последних изменений конфигурации в формате д.М.гггг.

Таблица отсортирована по имени конфигурации.

При наведении курсора на строку конфигурации появится кнопка для удаления
конфигурации: <img src={require('/img/rus/delete_btn.png').default} width="30"/>.

### Пользователи {#users}

Страница содержит список всех пользователей, зарегистрировавшихся в Системе или приглашенных в Систему,
а также возможность приглашения новых пользователей.

<p align="center">
<img src={require('/img/rus/users_page.png').default} width="900"/>
</p>

Чтобы пригласить нового пользователя в Систему, нажмите
на кнопку <img src={require('/img/rus/invite_user_btn.png').default} width="200"/>. В открывшемся окне
введите email пользователя, которого вы хотите пригласить в Систему:

<p align="center">
<img src={require('/img/rus/invite_user_window.png').default} width="450"/>
</p>

Также вы можете добавить для пользователя конфигурацию настроек, которая включает предустановки настроек
и/или шаблоны отчётов, которые будут добавлены в Систему пользователя при регистрации. То, как
добавить конфигурацию в вашу Систему, описано [выше](/overview/administration#configurations).

Наконец, вы можете определить роль приглашаемого пользователя:
- Администратор - пользователь, который может управлять работой в Системе других пользователей;
- Пользователь - пользователь, который
имеет все возможности работы в Системе, кроме взаимодействия с другими пользователями;
- Только чтение - пользователь с ограниченными правами: может только просматривать Программу, но
не может ей пользоваться.

После того, как вы добавили всю необходимую информацию,
нажмите на <img src={require('/img/runs/create_btn.png').default} width="170"/>. Ссылка для регистрации
нового пользователя будет отправлена ему на указанный email, а также скопирована в ваш буфер обмена, так
что вы можете поделиться ссылкой с пользователем напрямую.

Приглашенные или зарегистрировавшиеся в Системе пользователи представлены на странице в виде таблицы со следующими
колонками:
1. **ЛОГИН** - логин пользователя, указанный им при регистрации в Системе;
2. **EMAIL** - адрес электронной почты пользователя, использованный им при регистрации;
3. **РОЛЬ** - роль пользователя в Системе: Администратор, Пользователь или Только чтение.
Администратор может поменять роль других пользователей, нажав на текущее значение и выбрав другое.
4. **ВХОДИЛ В СИСТЕМУ** - дата (в формате д.М.гггг) и время (в формате чч:мм:сс) последнего входа пользователя в
Систему (ввода пары логин/пароль).

По умолчанию таблица отсортирована по логину пользователей. Её можно отсортировать по возрастанию или убыванию
значений всех её колонок.
Для сортировки таблицы по значениям колонки нажмите на её название (нажмите один раз, чтобы отсортировать
по возрастанию, или два раза, чтобы отсортировать по убыванию). После этого у названия колонки появится
стрелка <img src={require('/img/rus/sort_asc.png').default} width="17"/> (при сортировке по возрастанию)
или <img src={require('/img/rus/sort_desc.png').default} width="17"/> (при сортировке по убыванию).
Для сброса сортировки обновите страницу браузера. Колонки "*Логин*" и "*Email*" сортируются
по алфавиту, колонка "*Роль*" сортируется в следующем порядке: Пользователь, Только чтение, Администратор,
а колонка "*Входил в Систему*" - от наиболее ранних к наиболее поздним дате и времени и наоборот.

Нажав на строку своего пользователя, вы попадете в [свой профиль](/settings/profile).

При наведении курсора на строку другого пользователя появятся следующие
активные элементы: <img src={require('/img/rus/sing_as.png').default} width="30"/>, позволяющий
войти в аккаунт этого пользователя, и <img src={require('/img/rus/delete_btn.png').default} width="30"/>,
позволяющий удалить аккаунт пользователя из Системы. После нажатия на кнопку удаления необходимо будет
подтвердить действия в появившемся окне:

<p align="center">
<img src={require('/img/rus/delete_user.png').default} width="450"/>
</p>

Обратите внимание, что из Системы можно удалить аккаунты только неактивных пользователей, то есть
тех пользователей, у которых нет ни одного добавленного в Систему пациента или запуска.

### Активные задачи {#active-tasks}

Страница отображает, какие задачи процессинга образцов запланированы в данный момент у пользователей в Системе.

<p align="center">
<img src={require('/img/rus/active_tasks_page.png').default} width="900"/>
</p>

Информация на странице представлена в виде таблицы со следующими колонками:
1. **ПАЦИЕНТ/ЗАПУСК** - пациент и/или запуск, которому принадлежит обрабатываемый образец;
2. **ОБРАЗЕЦ/АНАЛИЗ** - название обрабатываемого образца;
3. **СТАТУС** - название запланированной задачи анализа и её прогресс;
4. **ЗАПЛАНИРОВАНО В** - дата (в формате д.М.гггг) и время (в формате чч:мм:сс), в которые было запланировано
проведение данной задачи;
5. **НАЧАТО** - дата (в формате д.М.гггг) и время (в формате чч:мм:сс) начала выполнения данной задачи.

Задачи группируются по пользователям, которым принадлежат обрабатываемые образцы.

Выполнение активной задачи можно отменить, нажав на <img src={require('/img/rus/cross_btn.png').default} width="20"/>
в её строке.
